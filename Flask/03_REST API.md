# Flask와 REST API

## REST API 개요

> **Re**presentational **S**tate **T**ransfer

### REST의 구성

- 자원(Resource)
- 행위(Verb)
- 표현(Representations)

### REST의 특징

1. Uniform Interface
2. Stateless
3. Cachable
4. Code-on-demand
5. Client-Server 구조
6. 계층형 구조

### REST API의 2가지 원칙

1. URI는 정보의 자원을 표현해야 한다.
2. 자원에 대한 행위는 HTTP Method(GET, POST, PUT, DELETE)로 표현한다.
